Коротка історія Git

Git створив Лінус Торвальдс у 2005 році. На той час команді, яка розвивала ядро Linux, потрібен був інструмент, що вміє швидко працювати з тисячами змін і не ламатися.

Звідси з'явилися основні ідеї Git:

  • зберігати знімки стану проєкту (коміти), а не просто різницю між файлами;
  • дозволяти легко повертатися назад, якщо щось пішло не так;
  • дозволяти працювати в окремих гілках, де можна сміливо експериментувати;
  • працювати навіть без інтернету, локально на вашому комп'ютері.

Для розробника Git — це «машина часу» для коду. Ви можете сміливо змінювати файли, знаючи, що завжди є точка, куди можна повернутися.

Сьогодні Git використовують практично всі: від студентських пет-проєктів до величезних корпоративних систем.
