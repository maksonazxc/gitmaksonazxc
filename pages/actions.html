Приклади основних команд Git

Нижче — невелика «шпаргалка» з найчастіших команд, які потрібні на старті.

Ініціалізація та базові налаштування

  • git init — увімкнути Git в поточній папці (створити репозиторій).
  • git config --global user.name "Ваше ім'я" — ім'я автора комітів.
  • git config --global user.email "you@example.com" — email автора комітів.

Додавання змін і коміти

  • git status — показує, які файли змінені/нові.
  • git add файл.txt — додати один файл до наступного коміту.
  • git add . — додати всі змінені файли.
  • git commit -m "Короткий опис змін" — зберегти знімок (коміт).

Робота з гілками

  • git branch — список гілок.
  • git switch -c new-feature — створити нову гілку і переключитися на неї.
  • git switch main — повернутися на основну гілку.
  • git merge new-feature — змерджити гілку new-feature у поточну.

Простий щоденний сценарій

  1. Зробіть зміни у файлах проєкту.
  2. Перевірте, що змінилось: git status.
  3. Додайте файли: git add ..
  4. Зробіть коміт: git commit -m "Коротко, що зробили".
  5. За потреби створіть гілку для нової фічі: git switch -c feature-idea.

Цих команд достатньо, щоб почати працювати з власними невеликими проєктами в Git.
