Спільна робота над кодом

Git дозволяє не лише зберігати історію свого проєкту, а й працювати разом з іншими людьми. Зазвичай це робиться через віддалений репозиторій на сервісах на кшталт GitHub або GitLab.

Віддалений репозиторій

Віддалений репозиторій — це копія вашого Git‑проєкту на сервері. Ви можете відправляти зміни командою git push і отримувати чужі зміни командою git pull.

Що таке fork і pull request

  • Fork — це ваша власна копія чужого репозиторію в обліковому записі GitHub.
  • Pull request (PR) — запит на те, щоб автор проєкту прийняв ваші зміни.

Простий план для першого внеску

  1. Знайдіть проєкт, який вам цікавий, і зайдіть у розділ Issues.
  2. Пошукайте задачі з міткою good first issue або beginner.
  3. Зробіть fork репозиторію та клонуйте свій форк собі на комп'ютер.
  4. Створіть гілку для своїх змін, виправте задачу і зробіть кілька комітів.
  5. Відправте гілку на GitHub (git push) і відкрийте pull request.
Внесок у спільні проєкти — це не тільки код. Ви можете допомогти з документацією, перекладом або прикладами.
